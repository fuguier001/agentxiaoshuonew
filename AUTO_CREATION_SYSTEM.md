- 节奏控制：小高潮 (10 章)、中高峰 (50 章)、大高潮 (100 章)

- 地图切换：新手村→州府→皇朝→大陆→上界→仙界

#### 单章创作 Prompt
- 开场 (300-500 字): 承接 + 场景 + 主角 + 冲突
- 发展 (1000-1500 字): 事件 + 互动 + 积累 + 波折
- 高潮 (800-1200 字): 爆发 + 出手 + 震惊 + 爽点
- 结尾 (300-500 字): 结果 + 收获 + 悬念 + 勾子

#### 伏笔管理 Prompt
- 短期伏笔 (1-10 章)
- 中期伏笔 (10-50 章)
- 长期伏笔 (100-500 章)
- 终极伏笔 (全书回收)

#### 节奏控制 Prompt
- 分析当前节奏位置
- 计算距离上次高潮的章数
- 给出本章节奏要求

#### 大模型遵循度优化 Prompt
- 明确指令重要性
- 惩罚机制说明
- 奖励机制说明
- 格式严格要求

## 专业性保证

### 1. 基于网文理论

#### 节奏控制理论
- 黄金三章 (1-3 章必须有爽点)
- 小高潮间隔 (10 章)
- 中高峰间隔 (50 章)
- 大高潮间隔 (100 章)

#### 爽点理论
- 打脸：被看不起→展示实力→震惊众人
- 装逼：低调→被迫出手→震撼全场
- 收获：战斗→胜利→获得宝物/功法
- 美人：邂逅→互动→收服
- 升级：瓶颈→突破→实力大增

#### 地图切换理论
- 新手村 (1-100 章)
- 州府 (101-300 章)
- 皇朝 (301-600 章)
- 大陆 (601-1000 章)
- 上界 (1001-2000 章)
- 仙界 (2001-3000 章)

### 2. 大模型遵循度优化

#### 问题
- 大模型经常忽略指令
- 输出格式不统一

#### 解决方案
1. **明确指令**: 清楚说明要求
2. **格式示例**: 提供 JSON 模板
3. **惩罚机制**: 说明错误后果
4. **奖励机制**: 说明完美效果
5. **分步执行**: 复杂任务分解

### JSON 解析

```python
def _extract_json(self, text: str) -> Dict:
    # 查找 JSON 起始和结束
    json_start = text.find('{')
    json_end = text.rfind('}') + 1
    
    if json_start >= 0 and json_end > json_start:
        json_str = text[json_start:json_end]
        return json.loads(json_str)
    else:
        raise ValueError("未找到 JSON")
```
